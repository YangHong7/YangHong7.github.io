一、ROM IP 配置

ROM功能的实现可以采用分布式内存（Distributed Memory）或者块内存（BlockMemory）

对比项 分布式内存 (Distributed Memory) 块内存 (Block Memory / BRAM)
物理底座 使用逻辑电路中的 LUT（查找表） FPGA 芯片中自带的专用存储块（Silicon Blocks）
适合场景 极小容量（如本题的 16 depth） 大容量、高带宽
资源消耗 消耗逻辑资源（LUTs），用多了会挤压逻辑空间 消耗芯片上固定的 BRAM 块（不占用逻辑空间）
灵活性 非常快，配置灵活 有固定格式，需按块使用

本实验数据量极小（只需要深度只有16位），故采用分布式内存。

IP Catalog

搜索框输入：Distributed Memory Generator 并双击

配置 memory config 标签页
Depth 设置为16，实际只需要12种状态
Data Width 设置为11 ，4位位选加7位段选

Port config：保持默认

配置 RST&Initialization 标签页

Load COE File 中上传编辑好的coe文件
COE 文件（Coefficient File）是 Vivado 中专门用来初始化存储器内容的文件

COE Options
Default Data 默认数值填0；Radix 填16 进制

OK、Generate并勾选global选项
